MVC Pattern Design

Dataflow
User (input) -> View (requests) -> Controller
Controller (responds) -> View (output) -> User

Controller (requests) -> DB Model
DB Model (responds) -> Controller

Controller (requests) -> AI Model
AI Model (responds) -> Controller

Startup
Controller asks to DB to open connection
Controller fetches from DB for past conversation
Controller sends to view to show past conversation

Shutdown
Controller asks DB to close connection
